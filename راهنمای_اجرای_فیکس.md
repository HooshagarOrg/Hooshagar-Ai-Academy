## 🔍 در صورت مشکل

### بررسی Manual:

```sql
-- 1. لیست کاربران بدون پروفایل
SELECT u.id, u.email, u.raw_user_meta_data
FROM auth.users u
LEFT JOIN public.profiles p ON u.id = p.id
WHERE p.id IS NULL;

-- 2. ایجاد دستی پروفایل برای یک کاربر خاص
INSERT INTO public.profiles (id, email, full_name, role)
VALUES (
  'USER_ID_HERE',
  'email@example.com',
  'نام کامل',
  'student'
)
ON CONFLICT (id) DO NOTHING;
```

## 📝 توضیحات فنی

### چی اتفاق افتاد؟

1. **Function**: تابعی که بعد از ایجاد user، خودکار profile می‌سازد
2. **Trigger**: روی جدول `auth.users` وقتی user جدید ساخته شود، function را اجرا می‌کند
3. **Policy**: اجازه می‌دهد کاربر خودش بتواند پروفایل ایجاد کند (برای امنیت بیشتر)
4. **Fix Existing**: همه کاربران موجود که پروفایل نداشتند، الان پروفایل دارند

### مزایا:

- ✅ Automatic: دیگه نیازی به ایجاد manual پروفایل نیست
- ✅ Secure: از metadata امن auth.users استفاده می‌کند
- ✅ Fast: در همان لحظه ثبت‌نام، پروفایل ایجاد می‌شود
- ✅ Backward Compatible: کاربران قدیمی هم فیکس شدند
