# الگوهای پیامک کاوه‌نگار — هوشاگر

**نوع استفاده برای همهٔ الگوهای production:** `وب سایت`  
(نه «تست فنی»، نه «سایر» — مگر واقعاً فقط تست موقت باشد.)

| فیلد فرم | مقدار پیشنهادی |
|----------|----------------|
| نام سامانه | `هوشاگر` |
| لینک سامانه | `https://www.hooshagar.ir` |
| نوع ارسال پیام | `پیامک` |
| ارسال مجدد با مسیر جایگزین | فعلاً غیرفعال |

> متغیرها فقط با `%token` / `%token2` / …  
> برای متن دارای فاصله (نام کامل) از `%token10` یا `%token20` استفاده کنید.  
> نام الگو باید **دقیقاً** با env یکی باشد (انگلیسی، بدون فاصله).

---

## ۱) ضروری الان — OTP ورود

| | |
|--|--|
| **نام الگو** | `otp-login` |
| **env** | `KAVENEGAR_TEMPLATE_OTP=otp-login` |
| **استفاده در کد** | `verify/lookup` در ارسال OTP |

**متن الگو:**
```
کد ورود شما به سامانه هوشاگر: %token%
این کد محرمانه است.
```

- `%token` = کد ۶ رقمی (بدون فاصله)

---

## ۲) پشتیبان OTP (اختیاری)

| | |
|--|--|
| **نام الگو** | `hooshagar-verify` |
| **env** | `KAVENEGAR_TEMPLATE_NAME=hooshagar-verify` |

**متن الگو:**
```
هوشاگر
کد تأیید شما: %token%
```

---

## ۳) نتیجه قرعه‌کشی

| | |
|--|--|
| **نام الگو** | `lottery-result` |
| **env** | `KAVENEGAR_TEMPLATE_LOTTERY=lottery-result` |
| **وضعیت کد** | فعلاً پیامک آزاد (`sms/send`)؛ الگو برای تأیید کاوه‌نگار / مهاجرت بعدی Lookup |

**متن الگو (پذیرش):**
```
هوشاگر
%token10% عزیز، در کلاس %token% پذیرفته شدید.
مدرسه: %token2%
```

- `%token10` = نام دانش‌آموز  
- `%token` = نام کلاس (بدون فاصلهٔ مشکل‌ساز؛ در صورت نیاز کوتاه کنید)  
- `%token2` = نام مدرسه  

**متن جایگزین (رد):** اگر کاوه‌نگار یک الگو برای هر دو حالت نمی‌پذیرد، الگوی دوم بسازید:

| نام الگو | `lottery-rejected` |
|----------|-------------------|

```
هوشاگر
%token10% عزیز، متأسفانه در قرعه‌کشی این دوره پذیرفته نشدید.
با مدرسه تماس بگیرید.
```

> الگوی فعلی `lottery-result` را ویرایش کنید: نام سامانه **هوشاگر**، لینک **`.ir`**, نوع استفاده **وب سایت**, و `{student}` را با `%token…` عوض کنید.

---

## ۴) حضور و غیاب (فعلاً ارسال خاموش)

| | |
|--|--|
| **نام الگو** | `attendance-notification` |
| **env** | `KAVENEGAR_TEMPLATE_ATTENDANCE=attendance-notification` |

**متن الگو:**
```
هوشاگر
اطلاع حضور: %token10% امروز %token% ثبت شد.
مدرسه: %token2%
```

- `%token10` = نام دانش‌آموز  
- `%token` = وضعیت (مثلاً غایب / تأخیر)  
- `%token2` = نام مدرسه  

---

## ۵) گزارش هفتگی (اختیاری — کد فعلاً متن آزاد می‌فرستد)

| نام الگو | `weekly-report` |

```
هوشاگر
گزارش هفتگی وضعیت آموزشی %token10% آماده است.
مشاهده در سامانه.
```

---

## ۶) دعوت والد (اختیاری)

| نام الگو | `parent-invite` |

```
هوشاگر
%token10% عزیز، ثبت‌نام انجام شد.
ورود به سامانه برای پیگیری وضعیت فرزند.
```

---

## اولویت ثبت در پنل

1. `otp-login` ← بدون این ورود با پیامک کار نمی‌کند  
2. اصلاح `lottery-result`  
3. `attendance-notification` (برای آینده)  
4. بقیه در صورت نیاز

پس از تأیید، در Vercel مقدار envها را با **نام دقیق** الگوها چک کنید (الان با `.env.local` هم‌خوان است).
