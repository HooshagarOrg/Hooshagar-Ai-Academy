## 🔐 **نحوه احراز هویت**

### **سیستم Authentication:**

```
Supabase Auth + JWT + Row Level Security (RLS)
```

### **فرآیند ورود:**

```
1. کاربر به /login می‌رود
2. ایمیل و رمز عبور وارد می‌کند
3. POST /api/auth/login ارسال می‌شود
4. Supabase Auth بررسی می‌کند
5. اگر صحیح: JWT Token صادر می‌شود
6. Token در httpOnly Cookie ذخیره می‌شود
7. Middleware نقش کاربر را چک می‌کند
8. Redirect به داشبورد مناسب
```

### **جدول‌های Database:**

```sql
-- 1. جدول احراز هویت (Supabase)
auth.users (
  id UUID,
  email TEXT,
  encrypted_password TEXT,
  raw_user_meta_data JSONB  -- {"role": "admin"}
)

-- 2. جدول پروفایل (Custom)
profiles (
  id UUID,              -- FK to auth.users
  email TEXT,
  full_name TEXT,
  role TEXT,            -- admin, teacher, parent, student, etc.
  school_id UUID,       -- FK to schools
  avatar_url TEXT,
  phone TEXT,
  created_at TIMESTAMP
)
```

## 🔒 **امنیت و دسترسی**

### **Row Level Security (RLS):**

```sql
-- مثال: معلم فقط کلاس خودش را می‌بیند
CREATE POLICY "teachers_see_own_classes"
ON students FOR SELECT
USING (
  class_id IN (
    SELECT id FROM classes 
    WHERE teacher_id = auth.uid()
  )
);

-- والد فقط فرزند خودش را می‌بیند
CREATE POLICY "parents_see_own_children"
ON students FOR SELECT
USING (parent_id = auth.uid());

-- دانش‌آموز فقط خودش را می‌بیند
CREATE POLICY "students_see_self"
ON students FOR SELECT
USING (user_id = auth.uid());
```
