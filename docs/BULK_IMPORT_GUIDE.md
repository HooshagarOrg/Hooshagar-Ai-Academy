# 📋 راهنمای ورود گروهی کاربران (نسخه ۲)

## مسیر UI

`/admin/bulk-import` — از منوی ادمین یا مدیر مدرسه: **واردسازی گروهی**

## فرمت فایل

یک فایل **Excel (.xlsx)** یا **CSV** با یک یا دو شیت:

### شیت ۱: دانش‌آموزان (+ والد در همان سطر)

قالب: `public/templates/bulk-import-students.csv`

| ستون | اجباری | توضیح |
|------|--------|-------|
| نام | ✅ | نام دانش‌آموز |
| نام_خانوادگی | ✅ | نام خانوادگی |
| کد_ملی | ✅ | ۱۰ رقم — کد ورود دانش‌آموز |
| پایه | ✅ | عدد یا فارسی (مثلاً هفتم) |
| کلاس | ❌ | نام کلاس |
| نام_والد | ❌ | نام والد |
| نام_خانوادگی_والد | ❌ | |
| کد_ورود_والد | ❌ | کد ملی والد (۱۰ رقم) |
| نسبت_والد | ❌ | پدر / مادر / سرپرست |
| موبایل_والد | ❌ | ۰۹... یا ۹... (۱۰ رقم بدون صفر) |

## مراحل واردسازی

1. مدرسه را انتخاب کنید
2. فایل را آپلود کنید (چند شیت خودکار تشخیص داده می‌شود)
3. پیش‌نمایش را بررسی کنید
4. **شروع واردسازی** — نتایج شامل کد ورود و PIN/رمز قابل دانلود است

## ورود بعد از import

| نقش | کد ورود | رمز |
|-----|---------|-----|
| دانش‌آموز | کد ملی ۱۰ رقمی | PIN ۴ رقمی (در نتایج import) |
| والد | کد ملی یا موبایل ۱۰ رقمی | ۴ رقم آخر کد (پیش‌فرض) |
| کارمند | کد ملی یا موبایل ۱۰ رقمی | ۴ رقم آخر کد (پیش‌فرض) |

صفحه ورود: `/login` — تب‌های کارکنان، والدین، دانش‌آموز، **پیامک**

### ورود با پیامک (OTP)
- تب **پیامک** یا لینک «ورود با پیامک» در هر تب
- شماره `09...` باید در `profiles.phone` ثبت شده باشد
- والدین و کارکنان: توصیه‌شده | دانش‌آموز: فقط با موبایل اختصاصی
- در dev، کد OTP در ترمinal سرور چاپ می‌شود (`[DEV] OTP for ...`)
