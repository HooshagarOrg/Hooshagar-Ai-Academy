## 👨‍👩‍👦 نسبت‌های معتبر (relation_type)

| نسبت | کد | استفاده |
|------|-----|----------|
| پدر | `father` | فقط برای `parent` |
| مادر | `mother` | فقط برای `parent` |
| سرپرست | `guardian` | فقط برای `parent` |
| کارمند | `staff` | برای `teacher`, `counselor`, etc. |

## ✅ قوانین مهم

### 1️⃣ **برای والدین:**
- `relation_type` اجباری است
- `student_national_code` اجباری است (برای پیدا کردن دانش‌آموز)
- هر دانش‌آموز می‌تواند حداکثر **2 والد** داشته باشد

### 2️⃣ **برای کارکنان (معلم، مشاور، ...):**
- `relation_type` خالی بگذارید یا `staff`
- `student_national_code` خالی بگذارید

### 3️⃣ **شماره موبایل:**
- فرمت: `09XXXXXXXXX` (11 رقم)
- یکتا باشد (تکراری نباشد)
- اگر خالی باشد، کاربر باید حضوری فعال‌سازی کند

### 4️⃣ **کد ملی دانش‌آموز:**
- 10 رقمی
- باید قبلاً در سیستم ثبت شده باشد

## 🚀 نحوه استفاده

### مرحله 1: آماده‌سازی فایل
1. فایل نمونه را دانلود کنید
2. در Excel یا Google Sheets باز کنید
3. ردیف‌های نمونه را حذف کنید
4. اطلاعات کاربران را وارد کنید

### مرحله 2: بررسی داده‌ها
- تمام ستون‌های اجباری پر شده باشد
- شماره‌های موبایل صحیح باشد (11 رقمی)
- کدهای ملی معتبر باشد (10 رقمی)
- نقش‌ها از لیست معتبر باشد

### مرحله 3: ذخیره فایل
- فرمت: CSV (UTF-8)
- نام فایل: `users_[تاریخ].csv`
- مثال: `users_1403-07-15.csv`

### مرحله 4: آپلود
1. وارد پنل ادمین شوید
2. **مدیریت کدهای فعال‌سازی** → **ورود گروهی**
3. فایل CSV را آپلود کنید
4. روی **پردازش و صدور کدها** کلیک کنید

### مرحله 5: دریافت کدها
- سیستم برای هر کاربر یک **کد فعال‌سازی یکتا** تولید می‌کند
- فایل Excel با کدها دانلود می‌شود
- شامل: نام، نقش، شماره موبایل، **کد فعال‌سازی**، **لینک فعال‌سازی**، **QR Code**

### مرحله 6: توزیع کدها
- فایل را چاپ کنید
- کارت‌های فعال‌سازی را بریده و به کاربران بدهید
- یا از طریق SMS ارسال کنید (اگر شماره موبایل دارند)
