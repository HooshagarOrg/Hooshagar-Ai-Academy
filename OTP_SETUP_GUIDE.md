# 📱 راهنمای راه‌اندازی OTP با کاوه‌نگار

---

## 🎯 **مراحل راه‌اندازی:**

### **مرحله 1: ثبت‌نام در کاوه‌نگار**

1. برو به [kavenegar.com](https://kavenegar.com)
2. کلیک روی "ثبت‌نام"
3. اطلاعات خود را وارد کنید
4. تأیید ایمیل و شماره موبایل

---

### **مرحله 2: دریافت API Key**

1. وارد پنل کاربری شوید
2. از منوی بالا: **تنظیمات** → **API Key**
3. کپی کردن API Key (مثال: `6B6C6D6E6F7071727374757677787980`)

---

### **مرحله 3: ایجاد Template برای OTP**

1. از منوی سمت راست: **خدمات** → **Lookup (کد تأیید)**
2. کلیک روی **ایجاد Template جدید**
3. پر کردن فرم:

```
نام Template: verify
متن پیام:
کد تأیید شما: {token}
اعتبار: 5 دقیقه
هوشاگر - hooshagar.ir
```

4. کلیک **ذخیره**

⚠️ **نکته مهم:** نام Template باید دقیقاً `verify` باشد (همان‌طور که در کد استفاده شده)

---

### **مرحله 4: افزودن API Key به پروژه**

1. باز کردن فایل `.env.local` در ریشه پروژه
2. افزودن خط زیر:

```env
KAVENEGAR_API_KEY=your-api-key-here
```

**مثال:**
```env
KAVENEGAR_API_KEY=6B6C6D6E6F7071727374757677787980
```

---

### **مرحله 5: اجرای Migration**

1. برو به **Supabase Dashboard**
2. از منوی سمت چپ: **SQL Editor**
3. کلیک **New Query**
4. باز کردن فایل `supabase/migrations/070_otp_authentication.sql`
5. کپی کردن تمام محتوا
6. پیست در SQL Editor
7. کلیک **RUN** ▶️

**انتظار خروجی:**
```
Success. No rows returned
```

---

### **مرحله 6: افزودن شماره موبایل به کاربران موجود**

اگر کاربران قبلی دارید، باید شماره موبایل آن‌ها را اضافه کنید:

```sql
-- در SQL Editor اجرا کنید:
UPDATE profiles
SET phone = '9123456789'  -- بدون صفر و +98
WHERE email = 'user@example.com';
```

**مثال برای چند کاربر:**
```sql
UPDATE profiles SET phone = '9123456789' WHERE email = 'admin@school.com';
UPDATE profiles SET phone = '9187654321' WHERE email = 'teacher@school.com';
UPDATE profiles SET phone = '9191234567' WHERE email = 'student@school.com';
```

---

### **مرحله 7: Restart Dev Server**

```bash
# در ترمینال:
Ctrl+C
npm run dev
```

## ✅ **تست عملکرد:**

### **تست 1: ارسال OTP**

1. برو به `http://localhost:3000/login`
2. کلیک روی تب **"ورود با پیامک"**
3. وارد کردن شماره موبایل: `09123456789`
4. کلیک **"ارسال کد تأیید"**

**انتظار:**
- ✅ پیام موفقیت: "کد تأیید به شماره موبایل شما ارسال شد"
- ✅ دریافت پیامک با کد 6 رقمی
- ✅ نمایش فرم وارد کردن کد
- ✅ شمارش معکوس 5 دقیقه

---

### **تست 2: تأیید OTP**

1. وارد کردن کد دریافتی (مثال: `123456`)
2. کلیک **"ورود به سیستم"**

**انتظار:**
- ✅ پیام موفقیت: "ورود موفقیت‌آمیز!"
- ✅ Redirect به داشبورد مربوط به نقش کاربر

---

### **تست 3: کد اشتباه**

1. وارد کردن کد اشتباه
2. کلیک ورود

**انتظار:**
- ❌ پیام خطا: "کد تأیید نامعتبر یا منقضی شده است"

---

### **تست 4: انقضای کد**

1. صبر کردن 5 دقیقه بعد از ارسال کد
2. تلاش برای ورود

**انتظار:**
- ❌ پیام خطا: "کد تأیید منقضی شده است. لطفاً کد جدید درخواست کنید."
- ✅ نمایش دکمه "ارسال مجدد کد"

## 🔍 **عیب‌یابی:**

### **مشکل 1: خطا "تنظیمات سرویس پیامک یافت نشد"**

**علت:** `KAVENEGAR_API_KEY` در `.env.local` تنظیم نشده

**راه حل:**
1. بررسی وجود فایل `.env.local` در ریشه پروژه
2. اضافه کردن خط: `KAVENEGAR_API_KEY=your-key`
3. Restart dev server

---

### **مشکل 2: خطا "کاربری با این شماره موبایل یافت نشد"**

**علت:** شماره موبایل در جدول `profiles` ثبت نشده

**راه حل:**
```sql
UPDATE profiles
SET phone = '9123456789'
WHERE email = 'your-email@example.com';
```

---

### **مشکل 3: پیامک ارسال نمی‌شود**

**بررسی‌های لازم:**

1. **API Key صحیح است؟**
   - ورود به پنل کاوه‌نگار
   - بررسی API Key

2. **اعتبار کافی دارید؟**
   - پنل کاوه‌نگار → موجودی
   - حداقل 100 تومان برای تست

3. **Template ایجاد شده؟**
   - پنل کاوه‌نگار → خدمات → Lookup
   - بررسی وجود template با نام `verify`

4. **شماره فرستنده فعال است؟**
   - پنل کاوه‌نگار → خطوط من
   - وضعیت: فعال ✅

---

### **مشکل 4: خطا در Console**

**بررسی لاگ‌ها:**

```bash
# در ترمینال dev server:
# جستجو برای:
"Kavenegar API error"
"Error sending OTP"
```

**لاگ‌های مفید:**
```
✅ OTP sent successfully: { phone: '9123456789', messageId: 123456 }

```

## 📊 **بررسی Database:**

### **چک کدهای OTP ارسال شده:**

```sql
SELECT 
  phone,
  code,
  verified,
  expires_at,
  created_at
FROM otp_codes
ORDER BY created_at DESC
LIMIT 10;
```

### **پاک کردن کدهای منقضی شده:**

```sql
SELECT cleanup_expired_otps();
```

### **بررسی کاربران با شماره موبایل:**

```sql
SELECT 
  id,
  full_name,
  email,
  phone,
  role
FROM profiles
WHERE phone IS NOT NULL;
```

## 🔐 **امنیت:**

### **محدودیت‌های پیشنهادی:**

1. **Rate Limiting:**
   - حداکثر 3 درخواست OTP در 10 دقیقه
   - حداکثر 5 تلاش اشتباه برای تأیید

2. **IP Blocking:**
   - مسدود کردن IP‌هایی با تلاش‌های مشکوک

3. **Phone Verification:**
   - تأیید شماره موبایل در زمان ثبت‌نام

## 💰 **هزینه‌ها:**

### **تعرفه کاوه‌نگار (تقریبی):**

- هر پیامک Lookup: **30 تومان**
- بسته 1000 پیامک: **25,000 تومان**
- بسته 10,000 پیامک: **220,000 تومان**

### **محاسبه هزینه ماهانه:**

```
فرض: 1000 کاربر فعال
میانگین ورود: 2 بار در روز
کدهای ارسالی در ماه: 1000 × 2 × 30 = 60,000

هزینه ماهانه: 60,000 × 30 تومان = 1,800,000 تومان
```
